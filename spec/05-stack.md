Стек — следствие принципов и предпосылка контрактов. Четыре слоя, у каждой строки есть причина.

## Слой 1 — ядро платформы

| Инструмент | Роль | Почему он |
|---|---|---|
| Python 3.12 | язык | требование задания |
| RabbitMQ 4.x + aio-pika | шина: доставка, очереди | штатные гарантии: подтверждение обработки, publisher confirms, повторная доставка, quorum-очереди с delivery-limit, dead-letter — отказоустойчивость конфигурируется, а не программируется; management UI; стандарт банковского enterprise. Точный образ актуальной поддерживаемой ветки пинится на S2 по release-информации |
| Pydantic v2 | контракты | конверт как типизированная модель, JSON-схема бесплатно; «контракт-труба» в коде |
| FastAPI | шлюз: HTTP-вход | Swagger UI из коробки — веб-интерфейс подачи заявок без строчки фронтенда |
| PostgreSQL 16 + asyncpg | склад состояния | требование «общие хранилища»; SQL руками, без ORM — состояния мало, магия не нужна |
| Qdrant (образ с пином версии) | склад векторов | общее хранилище обязано быть сервисом, не библиотекой; дашборд; паспорт коллекции — точкой id=0, без опоры на версионные фичи |

## Слой 2 — мозги PoC-агентов (сменное, не платформа)

| Инструмент | Роль | Почему он |
|---|---|---|
| LangGraph | граф рассуждения внутри executor | пожелание проверяющего «в идеале LangChain»; граф там, где есть ветвление (план → проверки → вердикт); parser и knowledge — прямые вызовы langchain-gigachat, граф из одного узла — золочение |
| langchain-gigachat | мост LangChain → GigaChat | официальный пакет Сбера; TLS — цепочка НУЦ Минцифры вшивается в образы агентов **и инструментов** (seed, smoke), не `verify=false` |
| GigaChat API | LLM | мозги в облаке — агенты локально невесомы; `base_url` конфигурируем: с 17.07.2026 действует `api.giga.chat` (сверено с документацией), прежний адрес выводится из эксплуатации |
| Эмбеддинги: GigaChat по API (дефолт) / bge-m3 через Ollama (офлайн) | текст → вектор для RAG | дефолт — эмбеддинги GigaChat по API: у корпоративного ключа доступны, путь лёгкий и быстрый. Офлайн-путь на локальной bge-m3 (Ollama-сервис, платформа ходит по HTTP из чертежа) — для ключей без доступа к эмбеддингам; образы агентов не тяжелеют, torch в них не живёт. Провайдер переключается строкой чертежа + переиндексацией — живая демонстрация паспорта коллекции (spec/45) |

## Слой 3 — наблюдаемость

| Инструмент | Роль | Почему он |
|---|---|---|
| structlog | структурные JSON-логи | логи фильтруются машиной, не глазами; trace_id в каждой записи |
| OpenTelemetry + Jaeger | трейсы с веб-UI | путь заявки через агентов виден как водопад; стандарт индустрии |

## Слой 4 — дисциплина разработки

| Инструмент | Роль | Почему он |
|---|---|---|
| Docker + docker compose | контейнеры, healthcheck, restart, одна команда | дословно из задания |
| uv | пакеты и окружения | быстрый современный стандарт |
| pytest + pytest-asyncio | тесты | гейты слайсов опираются на них; три яруса: unit / integration (сервис-контейнеры RabbitMQ·PG·Qdrant в CI, LLM и эмбеддер — детерминированные дубли с фиксированной размерностью) / e2e-live (живой GigaChat — только локально, `make test-e2e`) |
| ruff | линт и формат | чистота без ручной полировки |
| GitHub Actions | CI: линт + unit + integration | бесплатно для публичных репо; живого LLM в CI нет — зелёный бейдж повторяем |
| Makefile | команды в одно слово | проверяющий не запоминает длинное |

## Рассмотренные альтернативы

| Кандидат | Почему нет |
|---|---|
| n8n / Airflow / Temporal | центральный движок исполняет workflow, шаги — не независимые сервисы; нарушает требование «агенты — независимые сервисы» и проверяет умение установить, а не спроектировать |
| Kafka | конвейер для миллионов событий; у нас команды-и-ответы малого числа агентов — нужна семантика очередей с подтверждениями, не лог |
| Redis Streams | транспорт есть, но retry / dead-letter / маршрутизацию пришлось бы писать руками — самопис там, где есть штатное |
| NATS | лёгок и красив, но экзотика для российского enterprise — породит вопросы вместо доверия |
| «просто LangGraph» | граф внутри одного процесса; не даёт ни независимых сред выполнения, ни шины, ни платформы |
| Chroma | библиотека в процессе; общее хранилище обязано быть сервисом |
| pgvector | рабочий вариант «одним сервисом меньше», но выделенная векторная БД явно демонстрирует работу с векторным поиском и масштабируется отдельно от OLTP |
| BGE как sentence-transformers в процессе | тянет torch в каждый образ агента; принят вариант bge-m3 через Ollama-сервис — та же модель, но процессом сбоку, по HTTP |

# Бумага agentarium: порядок чтения и иерархия авторитетов

Здесь лежит истина архитектуры. Система сначала полностью написана на бумаге, потом реализована план-факт один к одному.

## Порядок чтения

Номер файла — это порядок. Читается сверху вниз, каждый следующий опирается на предыдущие:

```
00-principles.md      законы системы — что здесь нельзя нарушать
05-stack.md           инструменты и почему выбраны они
10-point-b.md         цель: что должно стать правдой
20-envelope.md        контракт конверта — язык, на котором говорят агенты
30-sdk.md             контракт разъёма — как агент подключается
40-topology.md        контракт чертежа — как система собирается из конфига
45-storage.md         контракт складов — состояние и знания
50-observability.md   контракт глаз — трейсы, логи, живучесть
55-poc-scenario.md    продуктовый сценарий примера (DBA-заявки)
60-acceptance.md      арбитр готовности: как проверить руками
70-roadmap.md         порядок стройки: слайсы и гейты
diagrams/             визуальные проекции спеков
```

## Шапка каждого спека

```
Статус: draft | accepted
Владелец: <какого знания этот файл — единственный дом>
Зависит от: <номера>
```

## Иерархия авторитетов

При конфликте документов побеждает верхний. Молча разойтись двум документам запрещено — младший чинится по старшему через правило врастания (`CLAUDE.md`).

```
PROGRAM_STATUS.yaml    факт: фаза, версия, что разрешено
   ▲
spec/00–70             истина: какой система обязана быть
   ▲
CLAUDE.md              процесс: как над ней работать
   ▲
docs/, spec/diagrams/  витрина: как её рассказывать
   ▲
код                    реализация: всегда чинится по спеке
```

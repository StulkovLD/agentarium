## Модель: шина, как в архитектуре ЭВМ

Модель платформы заимствована у компьютера: общая магистраль, к которой подключаются устройства. Устройства не знают друг о друге, у каждого есть адрес, протокол подключения един. Мы добавляем к железной модели одно — **очередь**: перед каждым агентом стоит буфер, и сообщение переживает падение получателя. Из этой одной добавки рождается вся отказоустойчивость.

**Опр.** Агент — независимый сервис в своём контейнере, подключённый к шине через SDK.
**Опр.** Конверт (Envelope) — единственная форма сообщения между агентами (`20-envelope.md`).
**Опр.** Маршрут (Route) — декларативное правило «тип сообщения → кому дальше» (`40-topology.md`).
**Опр.** Тип агента — код с логикой (класс). Экземпляр — тип, поставленный в систему с конфигом (объект). Один тип может стоять в системе много раз с разными конфигами.

## Три закона

1. **Ноль эвристик.** Смысл текста решает только LLM, детерминированное решает только код. Угадываний между ними нет: никаких `if "обнови" in text` в коде, никаких «модель сама разберётся» там, где нужен детерминизм.
2. **Один владелец на решение.** Каждый класс решения принимает ровно один узел:
   шина **доставляет** · маршруты **решают, кто следующий** · SDK **подключает** · агенты **думают**.
   Дубль-владельцев нет ни в системе, ни в коде, ни в бумаге.
3. **Контракт-труба.** Между узлами течёт типизированная структура — конверт, — а не сырой текст. Конверт собирает SDK, а не человек: протокол обеспечен кодом, не дисциплиной.

## Следствия законов

- **Шина тупая.** Интеллект по краям (агенты) и в декларации (маршруты). Логика в шину не затаскивается никогда — это могила Enterprise Service Bus, мы туда не идём. Формула: smart endpoints, dumb pipes.
- **Ядро вечное.** Ядро не знает ни предметной области, ни LLM-фреймворков. Добавление агента любого типа не требует изменения ядра: граница расширения системы — SDK.
- **Fail-fast.** Система, которая не может работать правильно, падает громко и сразу — на старте, с человеческим сообщением. Тихая деградация запрещена: мусорный результат, похожий на рабочий, хуже остановки.
- **Мозги — деталь агента.** LangGraph, голый вызов API, код без LLM — внутренность `handle()`, платформе безразлична. Зависимости мозгов изолированы в образе агента и физически отсутствуют в ядре.

## Границы (что платформой не является)

- Не оркестратор с центральным движком: нет процесса, который «исполняет workflow». Есть доставка, маршруты и агенты.
- Не фреймворк агентских мозгов: платформа не диктует, чем думать.
- Не продукт для конечного пользователя: инструмент инженера.

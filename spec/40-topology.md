# 40 · Чертёж системы

```
Статус: draft
Владелец: контракт конфигурации — реестр экземпляров и маршруты
Зависит от: 00, 20, 30
```

## Определение

**Опр.** Чертёж (topology) — YAML-файл, единственный источник состава системы: какие экземпляры агентов существуют и как связаны маршрутами. Ядро превращает чертёж в работающую систему.

## Формат

```yaml
# configs/dba-base.yaml
system: dba-requests

agents:                       # реестр экземпляров
  parser:                     # имя экземпляра
    type: parser              # тип из каталога агентов (образ)
    model:                    # конфиг мозгов — SDK передаст агенту как данные
      provider: gigachat
      name: GigaChat-2
  knowledge:
    type: rag
    collection: regulations   # какой коллекцией знаний пользуется этот экземпляр
    model: { provider: gigachat, name: GigaChat-2 }
  executor:
    type: executor
    target_db: env:TARGET_DB_DSN
    model: { provider: gigachat, name: GigaChat-2-Max }   # план работ — задача потяжелее

routes:                       # маршруты: тип конверта → очереди каких экземпляров
  request.new:      [parser]
  request.parsed:   [knowledge]
  knowledge.found:  [executor]
  plan.ready:       [gateway]
  task.failed:      [gateway]
```

Правила:
- **Тип ≠ экземпляр.** Один тип может стоять в системе много раз с разными конфигами: два `rag` с разными коллекциями — два разных узла из одного кода.
- Значение `env:ИМЯ` подставляется из окружения — секреты и адреса в чертёж не вписываются.
- Fan-out: несколько получателей у одного типа — конверт получает каждый: `plan.ready: [gateway, auditor]`.
- `gateway` — зарезервированное имя входа/выхода системы (`55-poc-scenario.md`).

## Как чертёж становится системой

1. **Маршруты = биндинги брокера.** Отдельного процесса-роутера нет. При старте рантайм объявляет topic-exchange `agentarium`, очередь `agentarium.<экземпляр>` каждому агенту и привязки «тип → очередь» по `routes`. Дальше доставляет сам RabbitMQ — машина маршрутов уже написана, мы её конфигурируем. Меньше движущихся частей — некому падать.
2. **Состав контейнеров генерируется из чертежа.** `make gen` собирает `docker-compose.agents.yml` из блока `agents` (образ по типу, env, healthcheck, restart policy). Инфраструктура (шина, склады, Jaeger, шлюз) — статичный `docker-compose.yml`. Одна правка чертежа — один файл, компоуз догоняет генератором.

## Валидация на старте — fail-fast

Рантайм отказывается стартовать с человеческим сообщением, если:
- маршрут ссылается на несуществующий экземпляр;
- тип конверта маршрутизирован агенту, чей манифест его не принимает (`consumes`);
- тип, который агент производит (`produces`), не имеет маршрута — конверту некуда лететь;
- у экземпляра типа `rag` не указана коллекция — и вообще при любом нарушении схемы чертежа (Pydantic).

Полусобранная система не поднимается. Либо чертёж цел, либо громкий отказ со списком дыр.

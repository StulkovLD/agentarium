Правила: один слайс — один пакет работ с гейтом. Слайс не закрыт — следующий не начат. Гейт = тесты зелёные + ревью diff + отметка в `PROGRAM_STATUS.yaml`. Каждый слайс — готовое ТЗ для агента-исполнителя.

---

## S1 · Скелет и конверт

- **Вход:** принятая бумага.
- **Работа:** каркас пакета (`uv`, `pyproject`, ruff, pytest, CI); Pydantic-модель конверта + генерация JSON-схемы; `Reply`.
- **Выход:** `core/envelope.py`, зелёный CI.
- **Тесты:** валидный/невалидный конверт, обязательность полей, сериализация туда-обратно, схема генерится.
- **Стоп:** ни строчки про шину, склады, агентов.

## S2 · Шина и SDK

- **Вход:** S1.
- **Работа:** клиент шины (aio-pika, quorum-очереди с delivery-limit, persistent + publisher confirms, порядок publish→confirm→ack, ретраи 1s/3s/9s, `task.failed`, DLX → dlq); класс `Agent` (петля жизненного цикла, манифест, схемы payload, `/health` = процесс+шина, вотчдог, graceful shutdown); echo-агент для тестов.
- **Выход:** `AGENT_INSTANCE=echo python -m agentarium run` живёт против RabbitMQ.
- **Тесты:** интеграционные с RabbitMQ как сервис-контейнером (бегут в CI): доставка, ack только после confirm, публикация в тип без маршрута → return-ошибка (mandatory), возврат неподтверждённого, ретраи, `task.failed` после исчерпания, delivery-limit → dlq, kill echo → конверт дожил.
- **Стоп:** без topology — биндинги в тестах руками; без LLM. Жертвуемое первым при срыве сроков: вотчдог → soft-timeout с громким логом.

## S3 · Чертёж

- **Вход:** S2.
- **Работа:** каталог типов + схема чертежа (Pydantic), загрузка + `env:` подстановка, валидации fail-fast рубежа 1 (все сценарии из 40), биндинги с реконсиляцией (diff: объявить/снять), генератор `docker-compose.agents.yml` из фиксированного шаблона, `make gen`.
- **Выход:** цепочка из двух echo-агентов собирается чертежом и гоняет конверт.
- **Тесты:** все fail-fast сценарии из `40-topology.md`, реконсиляция снимает биндинг-сироту, генератор идемпотентен, fan-out двум получателям.
- **Стоп:** без предметки. Страховка гейта: генератор compose съедает больше полудня — откат на статичный compose агентов, решение фиксируется в PROGRAM_STATUS.

## S4 · Глаза

- **Вход:** S2.
- **Работа:** structlog (JSON, обязательные ключи), OpenTelemetry в SDK, Jaeger в compose.
- **Выход:** путь конверта через двух echo — водопад в Jaeger, trace_id в каждой строке лога.
- **Тесты:** спан на обработку с атрибутами, trace_id прокинут между агентами.
- **Стоп:** без метрик-систем (Prometheus — лишнее для этого масштаба).

## S5 · Склады

- **Вход:** S3 (инжест читает секцию `collections` чертежа — нужен его лоадер).
- **Работа:** compose: Postgres + миграция requests, Qdrant (пин версии); `self.storage`; паспорт-точка id=0 + fail-fast сверка; `tools/ingest` + `make seed` (COLLECTION, вшит в `make up`).
- **Выход:** инжест markdown → поиск возвращает релевантный чанк.
- **Тесты:** интеграционные с сервис-контейнерами (бегут в CI): паспорт пишется/сверяется/отказывает; инжест идемпотентен; поиск с порогом, id=0 исключён.
- **Стоп:** содержимое `knowledge/` — заглушки из пары файлов; настоящие регламенты пишутся параллельно любому слайсу (тексты от кода не зависят) и финализируются к S7.

## S6 · Шлюз

- **Вход:** S3, S4, S5.
- **Работа:** FastAPI по HTTP-контракту из 40: `POST /requests` (запись + конверт `entry`-типа), `GET /requests/{id}`; консюмер финалов, исполняющий карту `finals` чертежа («первый выигрывает»); мини-консюмер `agentarium.dlq` (dead-lettered конверт → заявка `failed` по trace_id); проверка секретов на старте (fail-fast при плейсхолдере); healthcheck; вход в общий compose; `tools/smoke_gigachat.py` — ранний live-smoke (авторизация, чат-вызов, эмбеддинг) — TLS, ключ и квоты ловим здесь, а не в S7.
- **Выход:** end-to-end на echo-цепочке: POST → шина → финал → GET отдаёт результат; smoke зелёный на живом ключе. **Гейт G1: ядро построено.**
- **Тесты:** e2e happy path, e2e с `task.failed`, идемпотентность повторного финала, статусы заявки, 404.
- **Стоп:** шлюз не знает предметки: entry и финалы — только из чертежа.

## S7 · Агенты PoC

- **Вход:** G1 (+ регламенты `knowledge/`, писавшиеся параллельно).
- **Работа:** агенты parser (structured-вызов) / rag / executor (LangGraph, allowlist-проверки, read-only роль) по контрактам `55-poc-scenario.md`; target-db в compose с наполнением (роли, юзеры, read-only роль для executor); `configs/dba-base.yaml`.
- **Выход:** три демо-заявки (включая отказ `request.rejected`) проходят на живом GigaChat.
- **Тесты:** юнит на детерминированные части (allowlist-функции против target-db, сборка запроса rag) — в CI; контрактные на схемы выходов — в CI; e2e-live прогон трёх заявок — локально, `make test-e2e`.
- **Стоп:** ядро не трогается. Понадобилось тронуть — стоп, правило врастания.

## S8 · Демо и живучесть

- **Вход:** S7.
- **Работа:** тип `auditor` + `knowledge/incidents/` + `configs/dba-extended.yaml` (маршруты и финал переключены на аудит); `make demo CONFIG=…` (один прогон — одна конфигурация, красивый вывод: заявка → путь → ответ); сценарий kill-агента посреди прогона; `make up` полируется до одной команды (gen + up + wait-healthy + seed).
- **Выход:** сцена приёмки из `10-point-b.md` проходит целиком. **Гейт G2: PoC жив.**
- **Тесты:** приёмка `60-acceptance.md` по списку, руками + скриптом.
- **Стоп:** никакого фронтенда.

## S9 · Витрина и релиз

- **Вход:** G2.
- **Работа:** README финал (2 команды, сцена, скрин Jaeger); `docs/ARCHITECTURE.md` и `docs/GUIDE.md` дописаны по факту; диаграммы сверены с кодом; версии бампнуты; финальный прогон на чистом клоне.
- **Выход:** **Гейт G3: релиз.** Репо готов стать публичным; письмо для HR.
- **Стоп:** новые фичи запрещены. Только полировка и правда в документах.

**Release cut.** Обязательный минимум сдачи — S1–S7 + приёмка требований задания. Решение о cut — на гейте после S7, фиксируется в PROGRAM_STATUS.

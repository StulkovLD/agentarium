Правила: один слайс — один пакет работ с гейтом. Слайс не закрыт — следующий не начат. Гейт = тесты зелёные + ревью diff + отметка в `PROGRAM_STATUS.yaml`. Каждый слайс — готовое ТЗ для агента-исполнителя.

---

## S1 · Скелет и конверт

- **Вход:** принятая бумага.
- **Работа:** каркас пакета (`uv`, `pyproject`, ruff, pytest, CI); Pydantic-модель конверта + генерация JSON-схемы; `Reply`.
- **Выход:** `core/envelope.py`, зелёный CI.
- **Тесты:** валидный/невалидный конверт, обязательность полей, сериализация туда-обратно, схема генерится.
- **Стоп:** ни строчки про шину, склады, агентов.

## S2 · Шина и SDK

- **Вход:** S1.
- **Работа:** клиент шины (aio-pika, durable-очереди, ack, ретраи 1s/5s/25s, `task.failed`, dlq); класс `Agent` (петля жизненного цикла, манифест, `/health`, graceful shutdown); echo-агент для тестов.
- **Выход:** `python -m agentarium run echo` живёт против RabbitMQ.
- **Тесты:** интеграционные с живым RabbitMQ (compose): доставка, ack, возврат неподтверждённого, ретраи, `task.failed` после исчерпания, kill echo → конверт дожил.
- **Стоп:** без topology — биндинги в тестах руками; без LLM.

## S3 · Чертёж

- **Вход:** S2.
- **Работа:** схема чертежа (Pydantic), загрузка + `env:` подстановка, валидации fail-fast (маршрут в никуда, манифест-несовместимость, тип без маршрута), создание биндингов по маршрутам, генератор `docker-compose.agents.yml`, `make gen`.
- **Выход:** цепочка из двух echo-агентов собирается чертежом и гоняет конверт.
- **Тесты:** все fail-fast сценарии из `40-topology.md`, генератор идемпотентен, fan-out двум получателям.
- **Стоп:** без предметки.

## S4 · Глаза

- **Вход:** S2.
- **Работа:** structlog (JSON, обязательные ключи), OpenTelemetry в SDK, Jaeger в compose.
- **Выход:** путь конверта через двух echo — водопад в Jaeger, trace_id в каждой строке лога.
- **Тесты:** спан на обработку с атрибутами, trace_id прокинут между агентами.
- **Стоп:** без метрик-систем (Prometheus — лишнее для этого масштаба).

## S5 · Склады

- **Вход:** S1.
- **Работа:** compose: Postgres + миграция requests, Qdrant; `self.storage`; паспорт коллекции + fail-fast сверка; `tools/ingest` + `make seed`.
- **Выход:** инжест markdown → поиск возвращает релевантный чанк.
- **Тесты:** паспорт пишется/сверяется/отказывает; инжест идемпотентен; поиск с порогом.
- **Стоп:** содержимое `knowledge/` — заглушки из пары файлов, настоящие регламенты — S7.

## S6 · Шлюз

- **Вход:** S2, S3, S4, S5.
- **Работа:** FastAPI: `POST /requests` (запись + `request.new`), `GET /requests/{id}`; консюмер финалов (`plan.ready`, `task.failed`, `audit.done`) → результат в Postgres; healthcheck; вход в общий compose.
- **Выход:** end-to-end на echo-цепочке: POST → шина → финал → GET отдаёт результат. **Гейт G1: ядро построено.**
- **Тесты:** e2e happy path, e2e с `task.failed`, статусы заявки.
- **Стоп:** шлюз не знает предметки: финальные типы — из чертежа.

## S7 · Агенты PoC

- **Вход:** G1.
- **Работа:** написать регламенты `knowledge/` (6–8 + инциденты); агенты parser / rag / executor на LangGraph + langchain-gigachat по контрактам `55-poc-scenario.md`; target-db в compose с наполнением (роли, юзеры); `configs/dba-base.yaml`.
- **Выход:** три демо-заявки проходят цепочку на живом GigaChat.
- **Тесты:** юнит на детерминированные части (чек-функции executor против target-db, сборка запроса rag); контрактные на структуру выходов агентов; e2e прогон трёх заявок.
- **Стоп:** ядро не трогается. Понадобилось тронуть — стоп, правило врастания.

## S8 · Демо и живучесть

- **Вход:** S7.
- **Работа:** `configs/dba-extended.yaml` (auditor + fan-out); `make demo` (обе конфигурации, красивый вывод: заявка → путь → ответ); сценарий kill-агента в демо; `make up` полируется до одной команды.
- **Выход:** сцена приёмки из `10-point-b.md` проходит целиком. **Гейт G2: PoC жив.**
- **Тесты:** приёмка `60-acceptance.md` по списку, руками + скриптом.
- **Стоп:** никакого фронтенда.

## S9 · Витрина и релиз

- **Вход:** G2.
- **Работа:** README финал (2 команды, сцена, скрин Jaeger); `docs/ARCHITECTURE.md` и `docs/GUIDE.md` дописаны по факту; диаграммы сверены с кодом; версии бампнуты; финальный прогон на чистом клоне.
- **Выход:** **Гейт G3: релиз.** Репо готов стать публичным; письмо для HR.
- **Стоп:** новые фичи запрещены. Только полировка и правда в документах.

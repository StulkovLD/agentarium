# 20 · Конверт

```
Статус: draft
Владелец: контракт конверта — единственной формы сообщения между агентами
Зависит от: 00
```

## Определение

**Опр.** Конверт (Envelope) — типизированная структура, единственное, что летает по шине. Собирается SDK; агент руками конверт не строит никогда.

## Поля

| Поле | Тип | Кто заполняет | Смысл |
|---|---|---|---|
| `envelope` | const `"1"` | SDK | версия протокола; несовместимое изменение полей = новая версия |
| `id` | uuid | SDK | идентификатор конверта |
| `trace_id` | uuid | шлюз (при рождении заявки), дальше копируется | сквозной идентификатор заявки — нить, на которую нанизан весь путь |
| `causation_id` | uuid \| null | SDK | id конверта-причины; цепочка причинности для трейса |
| `ts` | ISO-8601 UTC | SDK | момент создания |
| `producer` | str | SDK | имя экземпляра-отправителя; только для наблюдаемости, для маршрутизации **не используется** |
| `type` | str | агент (возвратом из `handle`) | смысловой тип содержимого; единственное, по чему маршрутизируется конверт |
| `payload` | object | агент | содержимое; у каждого типа — своя Pydantic-схема (реестр схем — `30-sdk.md`), SDK валидирует вход и выход |
| `meta` | object, default `{}` | кто угодно | расширение без изменения протокола; ядро внутрь не заглядывает |

Правила:
- Адресата в конверте **нет** — это принцип: отправитель не знает получателя, маршрутизация декларативна (`40-topology.md`).
- Контекста трассировки в теле тоже **нет**: `traceparent` (W3C) живёт в AMQP-заголовках транспорта, владелец — SDK (`50-observability.md`). Конверт — бизнес-слой, склейка спанов — транспортный.
- `type` — нижний регистр, точечная нотация: `request.new`, `request.parsed`, `task.failed`.
- Реализация — Pydantic-модель в `core/`; JSON-схема генерируется из неё и лежит рядом с кодом. Схема — проекция, модель — истина.

## Ошибка — тоже конверт

Упавшая обработка не теряется и не пишется в пустоту. SDK публикует конверт `type: task.failed`:

```json
{
  "type": "task.failed",
  "payload": {
    "failed_type": "request.parsed",
    "reason": "GigaChat timeout after 3 retries",
    "attempts": 3,
    "original_payload": { "…": "…" }
  }
}
```

`task.failed` маршрутизируется как обычный конверт — в дефолтной конфигурации в шлюз (заявка завершается с честной ошибкой). Транспортные повторы до этого момента — зона SDK (`50-observability.md`).

```json
{
  "envelope": "1",
  "id": "0f2c7b1e-…",
  "trace_id": "9a41d6c0-…",
  "causation_id": "5d88a2f3-…",
  "ts": "2026-07-19T12:00:00Z",
  "producer": "parser",
  "type": "request.parsed",
  "payload": {
    "intent": "check_access",
    "entities": { "user": "ivanov", "database": "billing", "environment": "prod" }
  },
  "meta": {}
}
```

## Порядок работы

1. Бумага раньше кода. Имплементация ведётся только по спеке в статусе `accepted`.
2. Работа идёт слайсами из `spec/70-roadmap.md`. Один слайс — один пакет работ с гейтом.
3. Гейт слайса: тесты зелёные → ревью diff → отметка в `PROGRAM_STATUS.yaml`.
4. Больших взрывов нет: слайс не закрыт — следующий не начат.

## Правило врастания изменений

Изменение врастает, а не пришивается. Обнаружил, что спека неполна или неверна:

1. Стоп имплементация — код по устаревшей спеке не пишется.
2. Найди файл-владелец нового знания. Нет владельца — сначала реши, чей он.
3. Внеси изменение владельцу и пройди по всем задетым файлам: обнови ссылки, сними противоречия. Изменение встроено, когда карта снова цельная, а не когда абзац дописан.
4. Ревью по аксиомам: полнота, логичность, отсутствие лишнего, согласованность.
5. Обнови `spec/70-roadmap.md`, если сдвинулись слайсы; отметь в `PROGRAM_STATUS.yaml`.
6. Только теперь имплементация — план-факт один к одному.
7. Итерация закрыта (спека + код, оба отревьюены) — подними `version` в `PROGRAM_STATUS.yaml`.

## Камертоны кода

- Один модуль — один владелец решения. Модуль решает два вопроса — раздели.
- Между модулями — типизированные модели, не словари. Контракт-труба и в коде.
- Ноль эвристик: смысл текста решает только LLM, детерминизм — только код. Никаких `if "обнови" in text`.
- Fail-fast: не можешь работать правильно — падай громко и сразу. Тихий фолбэк запрещён.
- Имена — из словаря спеков: Envelope, Route, Agent. Код и бумага говорят одним языком.
- Нет класса там, где хватает функции. Нет абстракции «на будущее».
- Тесты проверяют контракты, не внутренности.
- Ошибка — это тоже конверт: структурная, с trace_id, а не print в пустоту.

## Коммиты

- Атомарный коммит = завершённый кусок слайса. В сообщении — ссылка на слайс: `S2: ...`.
- Сообщения по-русски, в повелительном наклонении: «добавь», «вынеси», «почини».
- Трейлер co-author обязателен — агентная разработка здесь не скрывается, а демонстрируется.

## Команды

| Команда | Что делает |
|---|---|
| `make up [CONFIG=…]` | вся система одной командой, строго в порядке: gen (файлы) → инфраструктура up + wait → topology apply (exchange/очереди/биндинги — брокер уже жив) → seed → агенты up (знания раньше агентов — иначе паспорт-сверка уронит rag). Compose собирается явно: `-f docker-compose.yml -f docker-compose.agents.yml` |
| `make gen [CONFIG=…]` | чертёж → валидация рубежа 1 + docker-compose.agents.yml. Только файлы: с живым брокером не разговаривает |
| `make demo [CONFIG=…]` | полный цикл: `make up CONFIG=…` + прогон демо-заявок. Требует живой ключ GigaChat. Смена CONFIG пересобирает систему, включая шлюз |
| `make seed [COLLECTION=…]` | проиндексировать базу знаний в Qdrant (по умолчанию все) |
| `make test` | тесты unit + integration (без живого LLM, эмбеддер-дубль) |
| `make test-e2e` | e2e на живом GigaChat (локально, не в CI) |
| `make lint` | ruff |

## Запреты

- В ядро (`core/`) не попадает предметка и не попадает LangChain. Мозги — дело агентов.
- Секреты не коммитятся. В git живёт только `env.example` с плейсхолдерами.
- Спека не меняется молча — только через правило врастания.
- Диаграммы — проекция спеков, не источник истины. При конфликте побеждает текст спеки.

## Definition of done слайса

Тесты зелёные + гейт-ревью пройден + `PROGRAM_STATUS.yaml` обновлён. «Скомпилилось» — не done.

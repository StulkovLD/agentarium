# Регламент резервного копирования

Заглушка слайса S5: настоящие регламенты пишутся к S7 (spec/70). Текст нужен лишь чтобы
инжест резал его на чанки по заголовкам, а поиск возвращал релевантный.

## Расписание бэкапов

Полный бэкап базы данных снимается еженедельно в ночь на воскресенье. Инкрементальные копии
делаются каждые сутки. Точка восстановления не старше двадцати четырёх часов.

## Хранение копий

Резервные копии хранятся тридцать дней в объектном хранилище с шифрованием. Копии старше срока
удаляются автоматически по политике жизненного цикла.

## Восстановление из копии

Восстановление проверяется ежемесячно тестовым прогоном на изолированном стенде. Ответственный
дежурный инженер фиксирует время восстановления в журнале.
